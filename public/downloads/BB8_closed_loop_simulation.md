# BB-8 阶段 10：编码器 / IMU 闭环运动仿真

> 结论：**软件闭环场景通过，真机门槛仍未通过。** 本阶段证明 C++ 控制核心在确定性双轮动力学模型中可以跟踪直线速度、完成约 90° 转弯，并在行驶中 IMU 数据过期时立即撤销电机输出。它不能替代编码器、IMU、电机、电池和球壳实测。

## 新增控制合同

- 200 Hz 控制周期，线速度与偏航率分别受 0.20 m/s²、0.80 rad/s² 斜坡限制。
- 左右轮目标由 `v ± ω·track/2` 生成，310 mm 轮距保持与 Blender 内部机构一致。
- 左右编码器轮速进入 PI 闭环；IMU 偏航率用于差动轮速修正。
- 编码器差速推导的偏航率与 IMU 偏航率差值超过 0.80 rad/s 时锁存 `motion_sensor_disagreement`。
- IMU 或任一编码器数据过期时锁存 `motion_sensor_stale`。
- 故障锁存后，当前周期左右 PWM 都归零；传感器恢复后仍需显式复位。

## 仿真模型

`tests/closed_loop_sim_test.cpp` 使用与生产固件相同的 `firmware/controller_core.cpp`，并加入：

- 左右电机 0.18 s 一阶惯性；
- 0.012 m/s 等效滚阻死区；
- 0.002 m/s 编码器量化；
- 310 mm 轮距差速运动学；
- 随轮速变化的简化 4S 电池压降；
- 与 PWM 平方相关的集总电机热模型；
- 2° 周期性底盘倾角扰动。

这些参数用于软件回归，不是从实物辨识得到的数字。

## 验收场景与结果

| 时间 | 场景 | 验收结果 |
| --- | --- | --- |
| 0–1.0 s | 静止安全启动 | 无故障、输出为零 |
| 1.0–4.5 s | 0.30 m/s 直线 | 稳态段 RMS 速度误差 0.00772 m/s |
| 4.5–7.1 s | 0.25 m/s、0.60 rad/s 转弯 | 实际转角 91.20° |
| 7.1–8.5 s | 斜坡停车 | 速度平滑归零 |
| 8.5–10.0 s | 0.20 m/s 再起步 | 故障注入前速度超过 0.10 m/s |
| 10.0 s | IMU 新鲜度失效 | 同周期 PWM 归零并锁存故障 |
| 10.8 s | 惯性滑行后 | 速度 0.00224 m/s |

完整 200 Hz 遥测位于 `engineering/closed_loop_telemetry.csv`，网站下载副本位于 `public/downloads/closed_loop_telemetry.csv`。

ESP32-S3 使用 Arduino-ESP32 3.3.10 重新编译通过：程序 338051 / 1310720 bytes（25%），全局变量 22492 / 327680 bytes（6%）。编译证明接口兼容，不证明传感器时序或电机实物安全。

## 真机前剩余门槛

1. 冻结编码器 PPR、边沿倍频、齿比与方向，使用架空轮实测每转计数。
2. 接入带时间戳的 IMU 姿态/角速度，并测量静止偏置、温漂和振动噪声。
3. 由实测阶跃响应辨识左右电机时间常数、死区和 PI 参数，禁止直接照搬仿真增益。
4. 增加驱动器电流采样、硬件限流和堵转检测。
5. 依次完成架空轮、开壳低压、封壳围栏、低速地面和热耐久试验。
