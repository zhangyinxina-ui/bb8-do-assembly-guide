# D-O 自组采购与调试路线

## 结论先行

当前已经具备一条可重复审计的 **D-O 控制系统路线**，但还不具备可合法再分发的完整机械包。能立即做的是：固定 v3.4.3 源码、准备控制台架、逐项验证接收机、IMU、声音和安全联锁。不能直接做的是：仅凭公开视频或控制仓库购买全部机械件、驱动电机和舵机，然后宣称一定能装成整机。

完整机械 STL/Fusion 仍指向 Mr Baddeley Patreon；购买属于外部付费动作，本项目不代购、不抓取、不绕过权限。公开的 60 mm STL 只是无轮调试支架，不能代替整机机械文件，而且未见明确再分发许可。

## 当前固定基线

- 本地仓库：`third_party/D-O-Printed-Droid`
- 固定提交：`e90aacdbe26a62fd4f0229d5504a3f2f3c409055`
- 推荐草图：`D-O_ibus_v3.4/D_O_printed_droid_rc_ibus_v3.4.3.ino`
- 控制器：Arduino Mega 2560
- 本地编译：Flash 43,198 / 253,952 B（17%），RAM 1,516 / 8,192 B（18%）
- 官方控制板页 12 个附件：本机已下载并哈希核验，保存在 Git 忽略目录；不在公开网站镜像
- AIO32 v2.1：实验性备选，22 个源码文件；ESP32 core 3.3.7 实编译通过（Flash 41%、RAM 15%），但无包级 LICENSE、无已找到的 KiCad/Gerber，且手册遗漏 SensorLib 依赖，不属于当前装机基线
- 机器可读采购清单：`engineering/do_self_build_bom.csv`

Mega 与 AIO32 编译通过只证明对应依赖和源码能生成固件，不证明真实电机、电池、机械结构或平衡控制已经安全。

## 先冻结的七个采购门

### G1：取得合法机械包

通过创作者授权渠道取得所需版本，并确认文件中包含哪些 STL、Fusion/STEP、BOM、打印方向和装配修订。付费访问不等于开源，也不自动授予再分发权。

**通过条件：** 文件版本、个人使用条款和完整清单已记录；缺件不再靠视频截图猜测。

### G2：解决驱动板冲突

仓库根 README 写的是 **一块 Cytron MDD10A 双路驱动**，v3.4 README 的硬件表写的是 **两块 MD10C**。两种方案都可能匹配草图的两组 PWM/DIR 输出，但接线、额定电流、散热和控制板布局不同。

**通过条件：** 根据实际控制板/接线图和电机峰值电流锁定一种架构；在此之前不购买驱动板。

### G3：测出驱动负载

公开控制仓库没有锁定驱动电机型号、减速比、轮径、整机质量、质心和真实堵转电流。README 中的电流数字只能作为来源记录，不能作为保险丝和驱动板选型依据。

**通过条件：** 实测空载电流、堵转/限流电流、轮端转矩、速度和热升；连续工况保留工程裕量。

### G4：测出四个舵机负载

README 同时写“4 个 standard servo”和“SG90 or similar”，这不是可直接采购的机械规格。主杆、头部俯仰/偏航/滚转的扭矩和行程必须从机械包量取。

**通过条件：** 每轴的静态/动态峰值扭矩、速度、角度、安装空间和供电电流已记录；舵机至少保留 2 倍实测扭矩裕量。

### G5：完成电源与安全设计

来源建议两块 2S LiPo 串联，形成 4S：16.8 V 满电、14.8 V 标称、12.0 V 空电。Mega、舵机、DFPlayer 和接收机不能直接共享未经设计的 4S 电源路径。

**通过条件：** 主保险、分支保护、硬件断电、逻辑稳压、舵机 BEC、导线截面积、连接器和散热均依据实测峰值电流确定。四个舵机不得由 Mega 5 V 引脚直接供电。

### G6：解决 Serial0 与 D0/D1 舵机复用

v3.4.3 源码把主杆舵机和头部舵机 1 固定到 D0、D1，同时执行 `Serial.begin(9600)` 并通过 Serial0 提供启动菜单、参数配置和测试命令。Mega 2560 官方引脚表把 D0/D1 定义为 RX0/TX0，因此这是同一对物理引脚的功能争用，不是可以忽略的文档差异。

**通过条件：** 固件与线束明确选择非 UART 舵机引脚，或采用经审查的隔离/复用设计；示波器/逻辑分析仪证明四路舵机脉宽、USB 上传、复位、9600 baud 菜单收发和失效保护均正常。在证据完成前，D0/D1 舵机信号保持断开，状态为 `HOLD_BENCH_VERIFICATION`。

### G7：冻结固件版本与接线图

Printed Droid 控制板页面内嵌的 **v2.1** 接线表把四个舵机列为 D2/D3/D4/D5；当前固定的 **v3.4.3** 源码和 GitHub README 则使用 D0/D1/D5/D6。页面的 `D-O_ibus_v3.4.zip` 也不是当前版本，其源码头为 v3.4.0。混用这些入口会把舵机接错，并掩盖 G6 的 Serial0 争用。

**通过条件：** 生成一份与实际编译文件绑定的线束合同，记录源码相对路径、Git 提交、源码 SHA-256、编译报告和四路舵机常量；上电前逐网导通确认。当前状态为 `HOLD_VERSIONED_WIRING_REQUIRED`，官网 v3.4 ZIP 只作历史参考，烧录输入保持 GitHub 固定的 v3.4.3。

## v3.4.3 接线合同

| Mega 引脚 | 功能 | 说明 |
|---|---|---|
| D0 / RX0 | 主杆舵机 | 与 Serial0 RX 复用；当前 HOLD，不接入首轮台架 |
| D1 / TX0 | 头部舵机 1 | 与 Serial0 TX 复用；当前 HOLD，不接入首轮台架 |
| D3 / D4 | PWM 驱动输入 | 仅 Mode 0 |
| D5 / D6 | 头部舵机 2 / 3 | 偏航 / 滚转 |
| D7 / D8 | DFPlayer RX / TX | Mega 直接控制声音 |
| D10 / D11 | 电机 2 PWM / DIR | 驱动板通道 2 |
| D12 / D13 | 电机 1 PWM / DIR | 驱动板通道 1 |
| D14–D17 | 四路声音开关 | 仅 Mode 0 |
| D19 | iBus RX | Mode 1；接收机与 Mega 共地 |
| D20 / D21 | SDA / SCL | IMU，400 kHz I2C |
| A15 | 电池采样 | 10 kΩ + 3.3 kΩ 分压 |

SBUS 也可进入 D19，但 Mega 不能自行反相，必须有外部硬件反相器。对新制作优先用 Mode 1 iBus，减少 PWM 线束；接收机必须真实提供 10 个通道，FS-iA6B 的 6 通道不足以覆盖默认完整映射。

以上表格记录的是固定 v3.4.3 源码现状，不表示 D0/D1 方案已经被工程接受，也不能用官网内嵌的 v2.1 D2–D5 表替换。官方 Mega 2560 串口映射见 https://docs.arduino.cc/language-reference/en/functions/communication/serial/ 。

## 分阶段调试路线

### D01 — 建立合法文件台账

**动作：** 记录机械包版本、许可证/个人使用条款、PDF、源码提交和每个文件哈希；把官网 12 个公开附件与 Patreon 付费文件分开登记。

**验收：** 所有付费与公开资源边界清楚；网站下载区没有再分发未授权 STL/Fusion。

### D02 — 冻结电气架构

**动作：** 依据实际控制板、接线图和电机峰值电流，解决 MDD10A 与 MD10C 冲突；画出电池、保险、断电开关、驱动、逻辑稳压和舵机 BEC 的单线图。

**验收：** 每条电源支路有电压、连续/峰值电流、线径、连接器和保护值；4S 满电 16.8 V 不会进入超额定电路。

### D03 — 无电机械干装

**动作：** 按授权机械包干装轮、轴承、主杆和头部机构，检查同轴度、间隙、重心和线束运动包络。

**验收：** 手动全行程无卡滞；紧固件可维护；线束不进入轮、齿轮和舵机臂扫掠区。

### D04 — 连续性与极性检查

**动作：** 在未接电池时逐网测量，核对公共地、正负极、分压器、保险和硬件断电。

**验收：** 无短路；所有连接器有标签和防反插方案；断电开关可从安全位置触达。

### D05 — USB 单板启动

**动作：** 只接 Mega USB，不接电机、电池和任何舵机；尤其保持 D0/RX0、D1/TX0 的舵机信号断开，重新编译并烧录固定 GitHub 提交中的 v3.4.3，不使用官网仍为 v3.4.0 的泛名 ZIP。

**验收：** 9600 baud 串口稳定显示版本；无两秒复位循环；上传的源码路径、提交、SHA-256、编译报告和版本化线束图已归档。

### D06 — IMU 单项验证

**动作：** 接入 IMU，运行菜单 `i` 检查轴向，再运行 `c` 或菜单 8 校准。

**验收：** WHO_AM_I 正确；前后倾与平衡角方向一致；静止零点稳定；I2C 400 kHz 连续运行无错误。

### D07 — 遥控链路验证

**动作：** 先打开遥控器，再启动控制器；菜单 `t` 查看所有通道，菜单 `p` 固定映射。

**验收：** `SIGNAL: OK`；CH1–CH10 逐项可识别；断开遥控后输入归中且操作员仍能硬件断电。

### D08 — 声音链路验证

**动作：** FAT16/FAT32 卡建立 `/mp3/0001.mp3` 至 `/mp3/0032.mp3` 的合法音频集合，接入 DFPlayer 与扬声器。

**验收：** 启动、警告和失联声音地址正确；最大所需音量无明显削波、复位或电源噪声。

### D09 — 舵机空载验证

**动作：** 舵机使用独立 BEC，只接一轴开始，逐步扩展到四轴；先限制角度与速度。D0/D1 两轴只有在完成 G6 的引脚改配或隔离方案后才可接入；所有四路信号必须先通过 G7 的版本化导通表。

**验收：** 四轴方向、零位、软限位正确；同步动作时舵机电源不跌落，Mega 不复位；USB 上传和 9600 baud 配置菜单仍可靠，且逻辑分析仪记录不存在 Serial0/舵机脉冲争用。

### D10 — 电机架空验证

**动作：** 将底盘固定在轮子离地的支架上，限流电源起步；用菜单 `m` 单独检查左右电机、方向交换和反向配置。

**验收：** 硬件断电立即让两电机失能；方向标记一致；空载电流和驱动温升已记录；手不进入轮子包络。

### D11 — A15 电池测量校准

**动作：** 接入 10 kΩ + 3.3 kΩ 分压，先用实验电源模拟电池电压，再启用监测。

**验收：** 满电、中点和低压三点与校准万用表误差在项目设定范围内；未接分压器时不启用电池监测。

### D12 — 低压与故障门

**动作：** 模拟临界电压、IMU 断线、遥控失联、超过 45° 倾倒和看门狗恢复。

**验收：** 行为与风险评估一致；每个故障的电机输出、声音和恢复条件均有记录。注意：源码设计中遥控失联仍可能继续平衡，不能把它当作完全断电。

### D13 — 系留首个平衡闭环

**动作：** 拆下头部并降低质心，使用两侧软系留和随手可达的硬件断电；从低输出限制和保守 PID 开始。

**验收：** 连续 30 秒不过流、不撞限位、不出现高频振荡；任何异常可在不靠近轮子的情况下断电。

### D14 — 装头与主杆耦合

**动作：** 逐项增加主杆和头部自由度，重新测量质心与平衡参数。

**验收：** 全姿态无机械干涉；舵机峰值电流、温升和回差记录完整；平衡控制不被头部动作击穿。

### D15 — 低速地面试验

**动作：** 在空旷区域，以低速度、低加速度完成直行、停止、转向和失联演练。

**验收：** 可预测停止；无电池压降复位；操作者始终保持安全距离并能远程/硬件切断。

### D16 — 热与耐久门

**动作：** 按真实任务循环运行，记录电池、电机、驱动器、稳压器、舵机和连接器温度。

**验收：** 在最高环境温度和最低电量下仍低于各器件额定限制；没有松动、磨线或连接器变色。

## 现在可买与暂缓购买

**可先买、风险较低：** Mega 2560、已确认逻辑电压的 MPU6050 模块、10 通道 iBus 接收机/遥控器、DFPlayer、microSD、8 Ω 小功率扬声器、精密 10 kΩ 与 3.3 kΩ 电阻、1 kΩ 电阻、合适耐压的 100 µF 电容、台架用线材和保险座。

**必须暂缓：** 驱动板、驱动电机/减速箱、四个最终舵机、最终 BEC/稳压器、LiPo 容量与 C 值、批量轴承/紧固件，以及任何声称包含完整 D-O 的非授权机械包。D0/D1 舵机线束或转接板也不得冻结，直到 G6 通过。

## 复核命令

```bash
python3 tools/audit_do_resources.py
python3 tools/audit_do_mantis_videos.py
tools/compile_do_firmware.sh
```

第一个命令核对仓库提交、许可证、PDF 哈希和机械文件缺口；第二个命令只读取 Mantis 播放列表元数据与描述链接，不下载视频；最后一个命令使用项目隔离的 Arduino 工具链重新编译 v3.4.3。
